Each Runset must have at least one Model.
Each Model is defined by:
  • Model label*:
    Short label describing the model. It must be unique within the new Runset.
  • Model description:
    Longer text in which the model may be described freely.
  • Hillslope model*:
    The vertical model that describes the flow of water within each hillslope.
    All hillslopes are going to be governed by the same Hillslope Model.
    The availability of a hillslope model depends on the initial conditions on the server.
  • Forcings*:
    Can be seen as the input data to be used in the model simulation.
    Each hillslope model requires a different set of forcings.
    The availability of a forcing source depends on the database condition in the server.
    Each forcing must have a source defined.
  • Global Parameters*:
    Each hillslope model requires a different set of global parameters.
    Each of then must have a value specified.
  • Evaluations:
    Each model output can be evaluated in different ways using different sources of data taken as reference.
  • Representations:
    Each hillslope model provides a different set of outputs.
    From each set of outputs a different set of representations can be obtained.
